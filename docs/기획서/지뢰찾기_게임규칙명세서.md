# 지뢰찾기 게임 규칙 명세서

## 1. 게임 개요

- **장르**: 퍼즐 (지뢰찾기)
- **플랫폼**: 웹 브라우저
- **목표**: 지뢰가 없는 모든 칸을 여는 것

## 3. 게임 규칙

### 3.1 보드 구성 (`sys_board`)

보드는 2차원 그리드(rows x cols)로 구성되며, 각 셀은 다음 상태 중 하나를 가진다.

#### 셀 상태

| 상태 | 설명 |
|------|------|
| `closed` | 닫힌 상태 (기본) |
| `opened` | 열린 상태 (숫자 또는 빈칸 표시) |
| `flagged` | 깃발이 꽂힌 상태 |
| `exploded` | 지뢰가 터진 상태 (게임 오버 시) |

#### 셀 내부 데이터

```
Cell {
  isMine: boolean        // 지뢰 여부
  adjacentMines: int     // 인접 지뢰 수 (0~8)
  state: CellState       // closed | opened | flagged | exploded
}
```

### 3.2 보드 생성 규칙

1. 플레이어가 **첫 번째 클릭**을 할 때 보드가 생성된다 (첫 클릭 안전 보장).
2. 첫 클릭한 셀과 그 **8방향 인접 셀**(최대 9칸)에는 지뢰를 배치하지 않는다.
3. 나머지 셀에 지뢰를 무작위 배치한다.
4. 각 셀의 `adjacentMines` 값을 계산한다 (8방향 인접 셀 중 지뢰 수).

### 3.3 입력 → 판정 → 결과

#### 좌클릭 (셀 열기)

| 조건 | 입력 | 판정 | 결과 |
|------|------|------|------|
| 셀이 `closed` + 지뢰 아님 + `adjacentMines > 0` | 좌클릭 | 숫자 셀 | 셀을 `opened`로 변경, 숫자 표시. `sfx_click` 재생 |
| 셀이 `closed` + 지뢰 아님 + `adjacentMines == 0` | 좌클릭 | 빈칸 셀 | 셀을 `opened`로 변경, **빈칸 연쇄 열기** 실행. `sfx_click` 재생 |
| 셀이 `closed` + 지뢰 | 좌클릭 | 지뢰 밟음 | 셀을 `exploded`로 변경, 모든 지뢰 공개, **게임 오버**. `sfx_explosion` + `fx_explosion` 재생 |
| 셀이 `flagged` | 좌클릭 | 무시 | 아무 동작 없음 (깃발 보호) |
| 셀이 `opened` | 좌클릭 | 무시 | 아무 동작 없음 |

#### 우클릭 (깃발 토글)

| 조건 | 입력 | 판정 | 결과 |
|------|------|------|------|
| 셀이 `closed` | 우클릭 | 깃발 설치 | 셀을 `flagged`로 변경. `ui_hud_minecount` -1. `sfx_flag` + `fx_flag` 재생 |
| 셀이 `flagged` | 우클릭 | 깃발 해제 | 셀을 `closed`로 변경. `ui_hud_minecount` +1. `sfx_flag` 재생 |
| 셀이 `opened` | 우클릭 | 무시 | 아무 동작 없음 |

#### 양클릭 / 더블클릭 (코드 열기, Chord)

| 조건 | 입력 | 판정 | 결과 |
|------|------|------|------|
| 셀이 `opened` + `adjacentMines > 0` + 인접 깃발 수 == `adjacentMines` | 양클릭 | 코드 열기 성공 | 인접한 `closed` 셀 모두 열기 (빈칸 연쇄 포함). `sfx_click` 재생 |
| 셀이 `opened` + `adjacentMines > 0` + 인접 깃발 수 != `adjacentMines` | 양클릭 | 코드 열기 실패 | 아무 동작 없음 |
| 코드 열기 시 잘못된 깃발이 있어 지뢰 셀이 열림 | 양클릭 | 지뢰 밟음 | **게임 오버** |

### 3.4 빈칸 연쇄 열기 (Flood Fill)

`adjacentMines == 0`인 셀을 열 때:

1. 해당 셀의 8방향 인접 셀 중 `closed` 상태인 셀을 모두 연다.
2. 열린 셀 중 `adjacentMines == 0`인 셀에 대해 재귀적으로 1번을 반복한다.
3. `adjacentMines > 0`인 셀은 열리지만 재귀하지 않는다 (숫자가 보이며 멈춤).
4. `flagged` 셀은 건너뛴다 (열지 않음).

**알고리즘**: BFS 또는 DFS (스택 오버플로우 방지를 위해 BFS 권장)

### 3.5 승리 조건

```
승리 = (열린 셀 수 == 전체 셀 수 - 지뢰 수)
```

- 모든 비지뢰 셀이 `opened` 상태이면 즉시 승리.
- 깃발을 모두 꽂을 필요 없음 (셀을 여는 것만 조건).
- 승리 시: `sfx_win` 재생, `ui_modal_result`에 "승리" + 소요 시간 표시, `sys_timer` 정지.

### 3.6 패배 조건

- 지뢰가 있는 셀을 열면 즉시 패배.
- 패배 시: 모든 지뢰 위치 공개, 잘못 꽂힌 깃발 X 표시, `ui_modal_result`에 "패배" 표시, `sys_timer` 정지.

## 4. 시스템 설계

### 4.1 게임 상태 머신 (`sys_game`)

```
[idle] ---(난이도 선택 또는 시작 클릭)--→ [ready]
[ready] ---(첫 번째 좌클릭)--→ [playing]
[playing] ---(지뢰 밟음)--→ [gameover]
[playing] ---(승리 조건 충족)--→ [victory]
[gameover] ---(재시작 클릭)--→ [ready]
[victory] ---(재시작 클릭)--→ [ready]
```

| 상태 | 설명 | 허용 입력 |
|------|------|----------|
| `idle` | 초기 화면 | 난이도 선택 |
| `ready` | 보드 표시, 타이머 미시작 | 좌클릭(첫 클릭), 우클릭 |
| `playing` | 게임 진행 중, 타이머 동작 | 좌클릭, 우클릭, 양클릭 |
| `gameover` | 패배, 입력 잠금 | 재시작만 가능 |
| `victory` | 승리, 입력 잠금 | 재시작만 가능 |

### 4.2 타이머 시스템 (`sys_timer`)

- 첫 클릭 시 시작 (0초부터)
- 1초 단위로 `ui_hud_timer` 갱신
- 최대 표시: 999초
- 게임 종료(승리/패배) 시 정지
- 재시작 시 0으로 초기화

### 4.3 남은 지뢰 카운터 (`ui_hud_minecount`)

```
표시값 = 총 지뢰 수 - 현재 깃발 수
```

- 음수 가능 (깃발을 지뢰 수보다 많이 꽂을 수 있음)
- 표시 범위: -99 ~ 999

### 4.4 점수 시스템 (`sys_score`)

| 항목 | 계산 |
|------|------|
| 기본 점수 | `(rows × cols - mines) × 10` |
| 시간 보너스 | `max(0, (난이도별_기준시간 - 소요시간)) × 5` |
| 최종 점수 | `기본 점수 + 시간 보너스` |

시간 보너스 기준:

| 난이도 | 기준 시간(초) |
|--------|-------------|
| 초급 | 60 |
| 중급 | 180 |
| 고급 | 480 |

## 5. 난이도 및 밸런싱

### 5.1 난이도 프리셋

| 난이도 | 행(rows) | 열(cols) | 지뢰 수(mines) | 지뢰 밀도 |
|--------|---------|---------|---------------|----------|
| 초급 (Beginner) | 9 | 9 | 10 | 12.3% |
| 중급 (Intermediate) | 16 | 16 | 40 | 15.6% |
| 고급 (Expert) | 16 | 30 | 99 | 20.6% |

### 5.2 커스텀 난이도 제한

| 파라미터 | 최소 | 최대 |
|---------|------|------|
| rows | 5 | 30 |
| cols | 5 | 50 |
| mines | 1 | `(rows × cols) - 9` |

- 지뢰 수 상한은 첫 클릭 안전 영역(9칸)을 보장하기 위함.

## 7. 인터페이스 명세 (개발팀 참조)

### 7.1 `sys_board` 인터페이스

```typescript
interface Board {
  rows: number;
  cols: number;
  mines: number;
  cells: Cell[][];

  generate(firstClickRow: number, firstClickCol: number): void;
  openCell(row: number, col: number): OpenResult;
  toggleFlag(row: number, col: number): FlagResult;
  chordOpen(row: number, col: number): OpenResult;
  isVictory(): boolean;
}

interface Cell {
  isMine: boolean;
  adjacentMines: number;
  state: 'closed' | 'opened' | 'flagged' | 'exploded';
}

type OpenResult = {
  success: boolean;
  openedCells: { row: number; col: number }[];
  hitMine: boolean;
};

type FlagResult = {
  success: boolean;
  newState: 'flagged' | 'closed';
  remainingMines: number;
};
```

### 7.2 `sys_game` 인터페이스

```typescript
interface GameManager {
  state: 'idle' | 'ready' | 'playing' | 'gameover' | 'victory';
  difficulty: Difficulty;
  board: Board;
  timer: Timer;

  startGame(difficulty: Difficulty): void;
  handleLeftClick(row: number, col: number): void;
  handleRightClick(row: number, col: number): void;
  handleChordClick(row: number, col: number): void;
  restart(): void;
}

type Difficulty = {
  name: string;
  rows: number;
  cols: number;
  mines: number;
};
```

### 7.3 `sys_timer` 인터페이스

```typescript
interface Timer {
  elapsed: number;  // 초 단위
  isRunning: boolean;

  start(): void;
  stop(): void;
  reset(): void;
  getElapsed(): number;
}
```

### 7.4 `sys_score` 인터페이스

```typescript
interface ScoreSystem {
  calculate(difficulty: Difficulty, elapsedSeconds: number): ScoreResult;
  getBestScore(difficulty: string): number | null;
  saveBestScore(difficulty: string, score: number): void;
}

type ScoreResult = {
  baseScore: number;
  timeBonus: number;
  totalScore: number;
};
```
